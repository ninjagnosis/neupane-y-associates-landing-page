# SEO Optimization & Custom 404 Page Design

**Date:** 2026-05-25  
**Project:** Neupane Y. & Associates website  
**Scope:** Comprehensive SEO overhaul + custom 404 error page

## Overview

Implement a complete SEO optimization strategy for the Neupane Y. & Associates chartered accountancy website, including:
1. Custom 404 error page with brand-consistent design
2. Technical SEO foundations (robots.txt, sitemap, structured data)
3. On-page SEO improvements (meta descriptions, heading hierarchy, keyword targeting)
4. Performance optimization verification

**Target audience:** Local and regional searches for accounting services in Nepal (Bhaktapur, Nepal region) + service-specific searches.

---

## 2. Technical SEO Layer

### 2.1 Robots.txt
**Location:** `public/robots.txt`

Create a basic robots.txt file that:
- Allows all search engines (`User-agent: *`)
- Disallows nothing (all content is indexable)
- Points to the sitemap: `Sitemap: https://neupaneassociates.com.np/sitemap-index.xml`

### 2.2 Sitemap
**Current state:** `@astrojs/sitemap` is already installed and configured in `astro.config.mjs`.

**Verification needed:**
- Confirm sitemap is generating correctly during build
- Ensure all pages are included (home, about, services, team, insights, contact)
- Verify sitemap-index.xml exists

**No additional code changes required.** The integration handles dynamic sitemap generation.

### 2.3 Structured Data (Schema.org JSON-LD)

Add schema markup to pages:

#### Homepage + Global
- **Organization** schema: name, logo, contact info, social profiles, founded date
  - Embed in layout or homepage component
  - Include: name, image, url, contactPoint (email, phone), address, sameAs (social links)

#### Homepage + About Page
- **LocalBusiness** schema: location (Bhaktapur), service area, hours of operation
  - Include: address, telephone, openingHoursSpecification, areaServed

#### Service Pages (each `/services/*/`)
- **Service** schema for each service type (Statutory Audit, Tax Advisory, etc.)
  - Include: name, description, provider (organization), areaServed

#### Insights / Blog Posts
- **BlogPosting** schema for each article in `/insights/`
  - Include: headline, description, image, datePublished, author

#### Navigation Structure
- **BreadcrumbList** schema on all pages (except homepage)
  - Shows: Home > Current Page hierarchy

**Implementation approach:** Create a reusable Astro component for each schema type, then include in appropriate layouts.

## 3. On-Page SEO Improvements

### 3.1 Meta Tags
- **Title tag:** 50–60 characters, keyword-first when relevant
  - Example: "Statutory Audit Services Nepal | Neupane Y. & Associates"
- **Meta description:** 150–160 characters, clear call-to-action where appropriate
  - Example: "Expert statutory audit and tax advisory services for Nepali businesses. Trusted by 40+ active clients."
- **Meta viewport & charset:** Already in place (verify)

### 3.2 Heading Hierarchy
- **H1:** One per page, descriptive and keyword-aware
  - Homepage: "Chartered Accountants for Ambitious Nepali Businesses"
  - Service pages: "Statutory Audit Services | Expert Accounting in Nepal"
- **H2–H3:** Logical structure, never skip heading levels
- **Avoid:** Multiple H1s, keyword stuffing

### 3.3 Content Optimization
Target search intents:

**Local searches** (e.g., "chartered accountants in Bhaktapur"):
- Ensure firm location (Bhaktapur) and service area are mentioned naturally in copy
- Use schema markup (LocalBusiness) for location prominence

**Service-specific searches** (e.g., "statutory audit Nepal"):
- Each service page targets its primary keyword
- Include keyword in title, H1, first paragraph, and naturally throughout
- Link related services internally

**General searches** (e.g., "accounting firm Nepal"):
- Homepage and About page target broader terms
- Cross-link to specific services from general pages

### 3.4 Image Optimization
- **Alt text:** Every image has descriptive, keyword-aware alt text
  - Example: "Yogesh Neupane, Chartered Accountant at Neupane Y. & Associates"
  - Not: "image1.jpg" or "picture"
- **File names:** Use descriptive names (already configured for Unsplash images)
- **Lazy loading:** Astro handles automatically

### 3.5 Internal Linking
- Link from service pages to related services
- Link from insights/blog posts to relevant service pages
- Use descriptive anchor text (never "click here")
- Ensure logical flow (no orphaned pages)
